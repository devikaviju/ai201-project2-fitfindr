### Tool 1: search_listings

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Searches the mock listings.json dataset for secondhand clothing items that match the user's provided description, size, and price constraints.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `description` (str): The search query or keywords describing the clothing item
- `size` (str): The size filter for the item
- `max_price` (float): The maximum price the user is willing to pay. Can be None.

**What it returns:**
<!-- Describe the return value — what fields does a result contain? -->
A list of dictionary objects representing matching items. Each dictionary contains fields like id, title, description, category, price, size, and brand.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if no listings match? -->
If no results match the constraints, the tool returns an empty list ([]) instead of throwing an error. The agent will then detect this empty list, halt the tool chain, and return a message asking the user to loosen their search constraints.

### Tool 2: suggest_outfit

**What it does:**
<!-- Describe what this tool does in 1–2 sentences -->
Uses an LLM to suggest a practical styling combination by pairing a newly found item with pieces from the user's existing wardrobe.

**Input parameters:**
<!-- List each parameter, its type, and what it represents -->
- `new_item` (dict): The specific item dictionary selected from the search_listings results.
- `wardrobe` (dict): The user's current wardrobe data, containing a list of items they already own.

**What it returns:**
<!-- Describe the return value -->
A string containing a description of how to style the new item with the existing wardrobe pieces.

**What happens if it fails or returns nothing:**
<!-- What should the agent do if the wardrobe is empty or no outfit can be suggested? -->
If the wardrobe dictionary is empty or minimal, the LLM prompt will fallback to providing general, trendy styling advice based purely on the new_item's characteristics so the application does not crash.

## Planning Loop

**How does your agent decide which tool to call next?**
<!-- Describe the logic your planning loop uses. What does it look at? What conditions change its behavior? How does it know when it's done? -->
The agent uses sequential conditional logic based on the outputs of each tool.

It first extracts parameters and calls search_listings.

It evaluates the result: If the results list is empty, it sets an error message in the session state and returns early (breaking the loop). If results are found, it proceeds.

It sets session["selected_item"] = results[0] and passes this to suggest_outfit.

It stores the generated outfit in session["outfit_suggestion"].

It passes both the outfit string and the selected item to create_fit_card to generate the final caption, storing it in session["fit_card"].

---

## State Management

**How does information from one tool get passed to the next?**
<!-- Describe how your agent stores and accesses state within a session. What data is tracked? How is it passed between tool calls? -->
State is tracked within a session dictionary that persists for the duration of the query.

Data found by search_listings is stored as session["selected_item"].

When calling suggest_outfit, the agent pulls session["selected_item"] to use as an argument.

The output is stored as session["outfit_suggestion"].

Finally, create_fit_card pulls both session["outfit_suggestion"] and session["selected_item"] directly from the session state dictionary to generate the final output.

For each tool, describe the specific failure mode you're handling and what the agent does in response.

| Tool | Failure mode | Agent response |
|------|-------------|----------------|
| search_listings | No results match the query |"I couldn't find anything matching that. Try removing the size filter or raising your budget!"|
| suggest_outfit | Wardrobe is empty |The llm generates a general styling advice rather than forcing a match with nonexistent pieces.|
| create_fit_card | Outfit input is missing or incomplete|"Oops, looks like the outfit context got lost. Try again!"|
